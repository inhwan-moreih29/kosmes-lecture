# 중소 제조업 AI 실습 강의 — 기획 메모

> 대상: AI 써봤지만 현장 적용은 아쉬웠던 제조 실무자 (수준 편차 1~3)
> 형식: 3타임 × 40분 / 노트북 + 간단 코딩

## 전체 흐름
1교시: **4종 비전 AI 실사용** (학습은 안 함, 허블로 직접 실습)
2교시: **비전 모델 영상 추론** (실시간 적용 → RISA)
3교시: **LLM 에이전트** (데이터 기반 의사결정 → M.AX)

- 도구: 1교시는 **허블 실제 UI로 직접 실습**. 제품 홍보톤 X, 자연스러운 도구로 사용.

## 1교시 (확정) — 4종 비전 AI 실사용

| 시간 | 내용 |
|---|---|
| 0–6 | 오프닝 + 4종 감 잡기 (분류/검출/분할/키포인트가 각각 뭘 푸는지) |
| 6–18 | 허블에서 4종 라벨링 + 추론 시각화 |
| 18–24 | 임계값 만지기 (오탐/미탐 트레이드오프) |
| 24–30 | 일부러 틀리기 (분포 밖 이미지) |
| 30–36 | 증강/하이퍼파라미터로 고치기 |
| 36–40 | 데이터 양 비교 + 핵심 3줄 정리 |

### 콘텐츠 메모
- **라벨링**: 4종 다 직접 시켜도 OK (스마트/오토 라벨링 가정 시 도구 익히기 포함 10~15분). 데이터셋 구축이 아니라 *체험*이 목적 → 늘어지지 않게 끊기.
- **임계값(추론 시점 노브)**: confidence 슬라이더 조절 → "불량 놓칠래 vs 정상 버릴래" 의사결정. 실시간 가능.
- **일부러 틀리기**: 조명/각도/처음 보는 불량 등 분포 밖 이미지 투입 → 모델이 틀리는 것 직접 관찰. "왜 내 AI가 별로였나"의 답.
- **증강/하이퍼파라미터(학습 시점)**: 그 자리 학습은 무리 → **강사가 미리 학습해둔 비교 모델을 추론으로 비교**.
  - 1순위 B: 증강 없음 vs 있음 → 위 "틀린 이미지"를 증강 모델로 다시 추론하면 맞음 (실패의 해결편)
  - A: 데이터 10장 vs 100장 (직관적, 임팩트 큼)
  - C(여유 시): 작은 모델 vs 큰 모델 = 속도 vs 정확도 → 2교시 복선
- **학습곡선(loss/정확도) 읽기**: 보너스 슬라이드로만 (편차 1번 청중엔 무거움).

### 마무리 멘트 (36–40, 제품 자랑 X · 체득 정리)
1. 라벨이 모델을 만든다 (알고리즘보다 데이터)
2. 판정선은 AI가 아니라 **내가** 정한다 (임계값)
3. 틀리는 건 AI가 멍청해서가 아니라 **데이터 분포 문제** → 증강·데이터로 고친다
- 다음 타임 복선: "이 모델을 영상에 실시간으로 돌리면?"

## 2교시 (거의 확정) — 비전 모델 영상 추론 + 전후처리

### 정체성
- 1교시 = 모델 그 자체 / 2교시 = **모델을 "시스템"으로** (영상 흐름 + 전후처리)
- 핵심 메시지: **"모델이 절반, 전후처리가 절반"** (특히 영상)
- 연결: 전후처리가 RISA의 본질 → 마지막에 "이게 현장에서 도는 게 RISA"

### 핵심 결정
- RISA UI엔 디바운싱/프레임스킵 등 노브가 없을 것 → **직접 만드는 교보재 앱**으로 진행 (제작은 직접)
- 프레이밍: "제품 내부를 까보는 학습용 도구" (RISA = 완제품, 앱 = 원리)
- **수강생이 직접 조작** → 짧은 데모(투어) + 긴 핸즈온 미션 구조
- 시간을 채우는 건 강사 설명이 아니라 **수강생이 직접 만지는 활동** (슬라이더 시연은 각 2~3분이면 끝)

### 노브 셋 (확정)
- **오탐 3종**: 신뢰도 임계값(출력 레벨, 1교시 복습) · 디바운싱(시간 레벨) · 최소 크기 필터(크기 레벨)
  - → "임계값·시간·크기 세 각도로 오탐을 깎는다"
- **속도 레버**: 프레임스킵 · 리사이즈 → FPS(실시간성) 트레이드오프 (1교시 "큰 모델은 느리다" 회수)
- **위험구역(ROI) 침입 판정**: 박스가 ROI 들어오면 경보 (RISA 직결)
- **추적 기반 분석**: 카운팅(몇 명) + 체류시간(얼마나 오래) — 둘 다 추적(ID 유지) 위에 얹힘, 한 코너로 묶기
- **(보너스)** 밝기/대비 보정 — 여유 시, 1교시 "분포 밖 실패" 회수
- 보조: 시각화 토글(박스/마스크/ID/confidence 표시)
- 뺀 것: ROI 크롭(위험구역과 혼동), 이동속도/라인계수/히트맵(욕심)

### 용어 메모
- **디바운싱**: 버튼 신호 안정화에서 온 말. 프레임마다 깜빡이는 탐지를 "N프레임 연속일 때만 인정" → 순간 오탐 제거. 대가는 반응 지연(트레이드오프).
- **추적/카운팅**: detection은 프레임마다 독립 → 추적으로 같은 객체에 ID 유지해야 "총 몇 명/체류시간"이 정확.

### 40분 배분 (직접 조작 전제)
| 시간 | 내용 |
|---|---|
| 0–5 | 도입: 파이프라인(입력→전처리→모델→후처리) + 앱 지도 |
| 5–16 | 강사 빠른 투어 — 노브 시연 (각 2~3분) |
| 16–35 | 핸즈온 미션 — "헛경보 없이 복합 규칙(동시 3명 초과 OR 10초 체류)이 제대로 울리게 노브 맞추기" |
| 35–40 | 정리 + RISA 연결 |

## 3교시 (확정) — LLM 에이전트로 데이터에 묻고 결정하기 (M.AX)

### 정체성
- 1·2교시 = **보는 AI(비전)** / 3교시 = **판단하는 AI(데이터)**
- 실제 제품 M.AX 사용 (실제 프로젝트: `~/workspaces/projects/max-agent`)
- M.AX = 사내 DB를 LLM이 읽기전용 쿼리 → 멀티턴 루프 → HTML 리포트로 답하는 에이전트
- 전체 강의 마무리: "AI에게 눈을 만들어주고(1·2교시) → 데이터를 읽는 머리를 붙인다(3교시)"

### 핵심 결정: 가상 회사 DB 사용 (직접 설계)
- 이유: ① 보안(실제 사내 DB 노출 회피) ② 통제(질문하면 인사이트 나오게 데이터 설계) ③ 공감(제조사 도메인) ④ 반복성
- **DB 붙이기 쉬움**: admin UI에서 데이터소스 등록(`POST /admin/data-sources`) → 30초 내 자동 인식, 재시작 불필요. 읽기전용 강제(SELECT만).
- 준비물: ① 가상 제조사 PostgreSQL(데이터 심기) ② 스키마 설명 문서(`detail_doc` 마크다운) — LLM은 자동 introspect 안 함, 사람이 써준 설명을 읽음

### 실행/배포: 서버형 다중 사용자
- max-agent는 backend SSE relay + worker 큐 구조 → **강사가 인스턴스 1개 띄우고 수강생은 브라우저 접속만** (환경 세팅 0)
- 사전 점검: 동시성 cap `WORKER_MAX_JOBS`(기본 10, 인원 많으면 상향) / OpenRouter API 비용·레이트리밋 / 모델(기본 Qwen 3.6 35B) 한국어 SQL 품질 테스트

### 교육 보너스 (제품에 이미 있는 것)
- **추론 trace 실시간 노출**: 어떤 쿼리를 왜 돌렸는지 다 보임 → "에이전트 머릿속 까보기" (1·2교시 "내부 까보기"와 일관)
- **스키마 설명이 답변 품질 좌우**: 부실 vs 충실한 설명 비교 = 실습 소재 (강사 데모)
- **읽기전용 가드레일**: "에이전트에 왜 안전장치가 필요한가"

### 남는 리스크
- 에이전트 비결정성(같은 질문에 답 흔들림/틀린 SQL) → **검증된 질문셋 + 답이 또렷하게 갈리는 데이터 설계**로 대응

### 가상 DB 설계 (예: 정밀부품 제조사)
- 테이블: `production`(라인별 생산/불량), `defects`(불량 유형·일자), `equipment`(설비 가동·다운타임), `sales`(수주·고객사·매출), `inventory`(재고)
- 심어둘 발견거리: 3번 라인 특정 주부터 불량 급증(설비 노후/자재 로트), 매출 한 고객사 편중, 재고 소진 임박 품목 1개

### 질문 흐름 (점층) + 미션
| 단계 | 질문 예시 | 보는 것 |
|---|---|---|
| 준비운동 | "지난달 라인별 생산량 보여줘" | sql_query trace + 표/차트 HTML |
| 분석(멀티턴) | "불량 제일 많은 라인은?" → "언제부터?" → "원인 후보는?" | 쿼리 연쇄 |
| 의사결정 | "경영진 보고용 Q2 품질 리포트 만들어줘" | write_artifact HTML 리포트 |
| 함정 | 데이터에 없는/모호한 질문 | 헛다리 → "근거(쿼리)를 봐라, 검증하라" |

- **핸즈온 미션**: "이 가상 회사에서 가장 시급한 품질 문제 1개를 찾아 경영진 리포트로 만들어라"

### 40분 배분
| 시간 | 내용 |
|---|---|
| 0–5 | 도입: "보는 AI → 판단하는 AI" + 에이전트 개념(LLM+도구+멀티턴) |
| 5–13 | 강사 시연: 단순 조회 → trace 까보기 → HTML 리포트 |
| 13–32 | 핸즈온 미션 (자유 질문 + 멀티턴 + 리포트, 중간에 함정 체험) |
| 32–40 | 정리: 좋은 질문·검증·가드레일 + "이게 M.AX" + 강의 전체 마무리 |
