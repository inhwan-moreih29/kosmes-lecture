회사 한빛정밀 (가상 정밀부품 제조사) DB PostgreSQL 16 (읽기전용, SELECT만) 테이블 16개 (마스터 6 · 운영 5 · 영업/재고 5) 기간 2025-01-01 ~ 2026-06-30 (18개월) · 기준일 2026-06-30 총 행수 약 16,000행 (생산 5,045 · 검사 4,480 중심) 설계 원칙 평범한 ERP/MES 스키마 — 정답을 테이블에 새기지 않고 조인으로 발견

설계 원칙 (먼저 합의할 것)

  • 안전 데이터를 특별 취급하지 않는다. RISA(2교시) 카메라가 뱉는 위험구역 경보도 safety_events라는 평범한 운영 로그 한 개로만 둔다. 스키마를 안전 중심으로 짜지 않고, 다른 운영 테이블과 동등하게 섞어 둔다 → MAX가 조인하다 "스스로" 연결을 찾게 한다.
  • 현실적인 제조사 스키마. 기준정보(고객·공급사·제품·라인·설비) + 트랜잭션(생산·검사·자재로트·설비정지·안전) + 영업/재고. 어디서나 보는 ERP/MES 형태.
  • 정답을 한 테이블에 적어두지 않는다. "라인3이 문제"라고 적힌 컬럼은 없다. 생산↔불량↔설비정지↔안전경보↔매출을 조인하고 시계열로 봐야 드러나게 데이터를 심는다.
  • 치수별 품번. 정밀부품답게 패밀리 + 치수 variant마다 고유 SKU(품번)를 부여 → 실제 제조사 품번 체계 느낌.

제품 / 품번 체계

한빛정밀은 5개 부품 패밀리를 생산. 같은 패밀리라도 치수가 다르면 다른 품번(SKU)이다. 품번 = 패밀리코드 - 주치수 - 부치수 [- 공차등급].

패밀리 코드   SH 정밀 샤프트 · BU 부싱 · PN 커넥터 핀 · GR 스퍼 기어 · VB 밸브 바디
품번 (part_no)패밀리치수공차등급현재 단가(원)

SH-08-060-IT6정밀 샤프트Ø8 × L60IT62,090

5개 패밀리 × 치수 variant = 총 14개 SKU (위는 대표 5개). 현재 단가는 products.unit_price, 과거 단가는 price_list(연 단위 개정), 주문 시점 단가는 order_items.unit_price에 스냅샷.

교육 포인트: 같은 "샤프트"라도 SKU가 여러 개 → "샤프트 매출"을 물으면 에이전트가 패밀리로 묶어야(GROUP BY family) 한다. 자연스러운 멀티턴/집계 소재.

관계 요약 (조인 경로)

products→product_families · equipment→production_lines · production_runs→(lines, products, material_lots) · quality_inspections→(production_runs, equipment) · equipment_downtime→equipment · safety_events→production_lines · material_lots→suppliers · sales_orders→customers · order_items→(orders, products) · inventory→products · price_list→products · inventory_snapshots→products

핵심: production_runs가 라인·제품·자재로트를 잇는 허브. 여기에 quality_inspections(품질)·equipment_downtime(설비, 라인 경유)·safety_events(안전, 라인 경유)·order_items(매출, 제품 경유)가 모두 시간축으로 만난다.

심어둔 발견거리 — "라인 3에 수렴"

정답을 적어두지 않고, 여러 테이블에 흩어진 신호가 2026년 4월 중순부터 정밀가공 3라인에 동시에 나타나게 심었다. 에이전트가 조인+시계열로 캐내야 보인다.

🎯 단일 루트 인사이트 (정답) 정밀가공 3라인의 노후 CNC 선반 3호(MORI SL-25, 2016년 설치)가 4월 중순부터 스핀들·주축 베어링 열화 → ① equipment_downtime 다운타임 급증 (3라인 월 비가동 ~480분 → 6월 2,494분) ② quality_inspections 치수공차이탈 불량 급증 (measured_value가 tol_lower~tol_upper 규격 이탈) — 불합격률 3월까지 0% → 4월 46% → 5·6월 100% ③ safety_events 라인3 위험구역 침입 경보 급증 — 작업자가 고장난 설비를 수동 개입하느라 진입 (2교시 RISA와 자연 연결) ④ 그런데 라인3은 최대 고객사(대성모터스, 매출 41%) 납품 샤프트(SH 패밀리) 주력 생산 → order_items 매출 편중 → 품질 문제가 핵심 매출을 위협
⚠ 미끼 (검증 교육용 함정) 같은 4월 중순, 라인3에 신규 공급사(대한특수강) 강재 로트가 처음 투입된다 → "자재가 불량 원인 아냐?"라고 오인하기 쉽게. 해소 단서: 같은 대한특수강 강재가 라인1에도 갔지만 거기선 불량이 0건(검사 72건/불합격 0) → 자재 무죄, 진짜 변수는 설비. "근거(쿼리)를 보고 검증하라"는 메시지를 데이터로 체험.

보조 미끼: 부싱 BU-16-10-IT7의 inventory.on_hand_qty(11,998)가 reorder_point(15,600) 밑 → "재고 소진 임박"이라는 곁가지 발견거리(메인 스토리와 무관, 자유 탐색용). 현재 재주문점 미달은 이 1개 품목뿐.
