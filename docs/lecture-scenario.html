우리 공장에 AI의 눈과 머리 달기

AI는 써봤지만 현장 적용은 아쉬웠던 제조 실무자가, 코딩 없이 비전 AI를 직접 다뤄보고(허블), 영상에 붙여 실시간 시스템으로 만들고(RISA), 데이터에 자연어로 물어 결정까지(M.AX) — 세 교시 내내 "내부를 까보고 직접 만지는" 핸즈온.

⏱ 3타임 × 40분 👥 AI 경험 有 · 수준 편차 1~3 💻 노트북 + 직접 조작 🏭 중소 제조업 현장

강의를 관통하는 그림 — 현장 AI의 두 축

👁

보는 AI — 비전

카메라 화면을 이해한다. 모델 자체를 다뤄보고(허블), 그 모델을 실시간 시스템으로 만든다(RISA).

세션 상세

1

모델 그 자체 — 4종 비전 AI 다뤄보기 허블

목표: AI를 갖다 쓰는 게 아니라 라벨링·추론을 직접 만지며 "AI가 어떻게 판단하고, 왜 틀리고, 어떻게 나아지는지"를 체득. (학습은 무거우니 미리 학습된 모델로 비교)

0–6오프닝 + 4종 감 잡기 분류/검출/분할/키포인트가 각각 뭘 푸는지. "왜 내 AI는 별로였나"로 문제 제기.
6–18허블에서 4종 라벨링 + 추론 시각화 각 태스크 10장 내외 라벨링 후, 출력 형태(라벨/박스/마스크/점) 비교.
18–24임계값 만지기 신뢰도 슬라이더 → "불량 놓칠래 vs 정상 버릴래" 오탐/미탐 의사결정.
24–30일부러 틀리기 조명·각도·처음 보는 불량 등 분포 밖 이미지 → 모델이 틀리는 것 관찰.
30–36증강/하이퍼파라미터로 고치기 미리 학습해둔 비교 모델(증강 유무·데이터 양)로 "틀린 걸 고치는" 해결편.
36–40핵심 3줄 정리 데이터가 성능을 만든다 · 판정선은 내가 정한다 · 틀림은 분포 문제. → 다음 타임 복선.
핵심 실습 ① · 허블 직접 실습 (난이도 ★☆☆~★★☆)
이미지→라벨링→ 추론·시각화→임계값→ 실패→증강→데이터양 비교
  • 오탐을 다스리는 감각: 임계값을 밀며 검출이 늘고 주는 트레이드오프 체감
  • 실패의 정체: 분포 밖 이미지로 틀려보고 → 증강 모델로 다시 추론하면 맞음 (해결편)
  • 데이터의 힘: 10장 vs 100장 모델 비교 (알고리즘보다 데이터)
산출물: 4종 출력 형태 비교 경험 + "우리 공정 어디에 어떤 비전 AI를 쓸까" 메모
2

모델을 시스템으로 — 영상 추론 + 전후처리 RISA

목표: 1교시의 모델이 영상에선 전후처리가 붙어야 쓸모가 된다. 노브를 직접 밀어 트레이드오프를 체감. "모델이 절반, 전후처리가 절반."

0–5도입: 파이프라인 + 앱 지도 입력→전처리→모델→후처리→판정. "제품 내부를 까보는 학습용 도구"임을 명확히(≠RISA 그 자체).
5–16강사 빠른 투어 노브를 각 2~3분씩 시연 — 이게 뭘 바꾸는지.
16–35핸즈온 미션 수강생이 직접 노브를 조합해 목표를 맞춤(아래).
35–40정리 → RISA 연결 "이 원리들이 제품 안에 세팅돼 24시간 도는 게 RISA." (여유 시 추적/카운팅 클립)
핵심 실습 ② · 직접 만드는 교보재 앱 (수강생 직접 조작)
  • 오탐 3종: 신뢰도 임계값(출력) · 디바운싱(시간) · 최소 크기 필터(크기) — "세 각도로 오탐을 깎는다"
  • 속도 레버: 프레임스킵 · 리사이즈 → FPS(실시간성) 트레이드오프
  • 위험구역(ROI) 침입 판정: 박스가 구역에 들어오면 경보 (RISA 직결)
  • 추적 기반 분석: 카운팅(몇 명) + 체류시간(얼마나 오래) — 추적 ID 위에 함께
  • (보너스) 밝기/대비 보정 — 1교시 "분포 밖 실패" 회수
🎯 미션: 이 영상에서 헛경보 없이 복합 규칙(위험구역 동시 3명 초과 또는 한 명이라도 10초 이상 체류)이 제대로 울리도록 노브를 맞춰라.
산출물: 직접 튜닝한 실시간 위험 탐지 설정 + "우리 현장 어느 구역에 어떻게 적용할까" 메모
3

데이터에 묻고 결정하기 M.AX

목표: 보는 AI(비전)에서 판단하는 AI(데이터)로. 가상 회사 DB에 자연어로 질문하면 에이전트가 스스로 쿼리·분석하고 HTML 리포트로 답한다. 추론 과정을 까보며 검증하는 자세까지.

0–5도입 "보는 AI → 판단하는 AI". 에이전트 개념 = LLM + 도구(쿼리) + 멀티턴 루프.
5–13강사 시연 단순 조회 → 추론 trace 까보기(어떤 쿼리를 왜 돌렸나) → HTML 리포트 한 번.
13–32핸즈온 미션 자유 질문 + 멀티턴으로 파고들기 + 리포트. 중간에 함정(검증) 체험.
32–40정리 + 강의 전체 마무리 좋은 질문·검증·가드레일 + "이게 M.AX" + 눈+머리로 닫기.
핵심 실습 ③ · 가상 회사 DB + M.AX (브라우저 접속)
자연어 질문→에이전트가 쿼리→ trace 관찰→멀티턴 분석→HTML 리포트
  • 점층 질문: 단순 조회 → "불량 1위 라인은?→언제부터?→원인은?"(멀티턴) → 경영진 리포트
  • 함정 체험: 데이터에 없는/모호한 질문 → 헛다리 → "근거(쿼리)를 봐라, 검증하라"
  • 가드레일: 읽기 전용(SELECT만) — "에이전트엔 안전장치가 필요하다"
🎯 미션: 이 가상 회사에서 가장 시급한 품질 문제 1개를 찾아내 경영진 리포트로 만들어라.
산출물: 자연어로 발굴한 인사이트 + 직접 만든 HTML 리포트 + "내 업무에서 물어보고 싶은 질문" 리스트

준비 현황 — 준비된 것 vs 준비할 것

✅ 이미 준비된 것 (기존 제품·인프라)

  • ✓T1
    허블 — 노코드 비전 AI 학습 플랫폼. 실습 플랫폼으로 그대로 사용
  • ✓T2
    RISA — 산업안전 실시간 탐지 제품. 완성 결과 시연 + 연결용
  • ✓T3
    M.AX (max-agent) — DB 에이전트 제품. 기능 대부분 완성
  • ✓T3
    다중 사용자 서버 구조 — 강사 1인스턴스 + 수강생 브라우저 접속 가능
  • ✓T3
    가상 DB 연결 기능 — admin UI 등록 → 30초 내 인식, 읽기 전용 강제
  • ✓T3
    추론 trace 노출 · HTML 리포트 생성 — 교보재로 바로 활용

🛠 준비할 것 (강의용 제작·세팅)

  • ▢T1
    샘플 부품 이미지셋 (정상/불량, 가명화) + 분포 밖 실패용 이미지
  • ▢T1
    미리 학습된 비교 모델 (증강 유무 · 데이터 10장/100장)
  • ▢T1
    허블 강의용 계정/샌드박스 확보
  • ▢T2
    전후처리 교보재 앱 제작 (노브 셋) + 백업 녹화 클립
  • ▢T2
    샘플 CCTV 영상 + 수강생 배포 방식(실행파일 등)
  • ▢T3
    가상 제조사 DB 설계 + 발견거리 심은 데이터
  • ▢T3
    스키마 설명 문서(detail_doc) 작성 — 답변 품질 좌우
  • ▢T3
    검증된 질문셋 (비결정성 대응) + 인스턴스 점검(동시성/비용/모델)
  • ▢공통
    핸즈온 미션 지문 + 수준별 완성본/힌트(편차 1~3 대응)

운영 가이드

🎚 수준 편차(1~3) 대응

  • 직접 조작 중심: 설명 최소화, 손이 시간을 채우는 핸즈온 미션 구조
  • 완성본/힌트 미리 배포로 막혀도 흐름 유지
  • 심화 과제를 각 타임에 둬서 빠른 사람 공백 방지
  • 버디 멘토: 빠른 수강생을 옆자리 도우미로

⚠ 라이브 리스크 체크

  • T2 앱: 강의장에서 죽을 때 대비 백업 녹화 클립 필수
  • T3 에이전트: 비결정적 → 검증된 질문셋, 데이터를 답 또렷하게 설계
  • T3 동시성: WORKER_MAX_JOBS 인원 맞춰 상향, API 비용·속도 사전 점검
  • 무거운 추론/학습은 미리 돌려두거나 강사 데모로

한 줄 메시지 (강의 관통 주제)

  • TIME 1 — "AI가 어떻게 판단하고 왜 틀리는지, 직접 만져보면 보인다." → 허블
  • TIME 2 — "모델이 절반, 전후처리가 절반. 시스템이 돼야 현장을 지킨다." → RISA
  • TIME 3 — "보는 AI를 넘어 데이터에 묻고 결정한다 — 단, 근거를 보고 검증하며." → M.AX
  • 마무리 — "AI에게 눈을 만들어주고(1·2교시) 데이터를 읽는 머리를 붙인다(3교시)."
