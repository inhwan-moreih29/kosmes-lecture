2교시 강의 시나리오 (내부 논의용 초안)

중소제조업 비전 AI 실습 · "모델이 절반, 전후처리가 절반" · 영상 추론 교보재 앱 핸즈온 → RISA 연결

대상 중소제조업 실무자 시간 40분 (3타임 중 2타임) 도구 직접 만든 교보재 앱 (PySide6) 핵심 "모델은 1교시에 봤다 — 영상에선 전후처리가 절반"
파이프라인·앱 지도→ 노브 투어→ 핸즈온 미션→ RISA 연결

0–5 도입 — 파이프라인 + 앱 지도

1교시는 모델 그 자체(정지 이미지·임계값)였다. 2교시는 그 모델을 영상에 흘려보내 "시스템"으로 만든다. 화면 한 장으로 파이프라인을 박는다: 입력 → 전처리 → 모델 → 후처리 → 판정.

멘트: "모델은 박스를 뱉을 뿐이다. '3명이 위험구역에 10초 있으면 경보' 같은 현장의 판단은 전부 전후처리가 만든다. 오늘 만질 노브가 바로 그 절반이다."

앱 위치 설정: 이건 RISA가 아니다. "제품 내부를 까보는 학습용 도구" — RISA는 완제품, 이 앱은 그 안에서 도는 원리를 손으로 만지게 노출한 것. 왼쪽 영상 뷰 + 오른쪽 노브 패널 구조를 한 번 훑는다.

5–16 강사 빠른 투어 — 노브가 뭘 바꾸나 시연

각 노브를 2~3분씩, "끄고 켜서 화면이 어떻게 변하나"만 보여준다. 설명보다 직접 밀어 보이는 게 핵심. 노브는 세 묶음.

묶음 ① 오탐을 깎는 3종 — 출력·시간·크기
신뢰도 임계값
출력 레벨. 1교시 복습. 낮추면 다 잡지만 헛검출↑, 높이면 깔끔하지만 놓침↑.
디바운싱
시간 레벨. "N프레임 연속으로 잡혀야 인정" — 한 프레임이라도 놓치면 카운트 리셋. 순간 깜빡임(오탐) 제거, 대가는 반응 지연.
최소 크기 필터
크기 레벨. 박스 높이가 화면의 N% 미만이면 버린다. 멀리 작게 잡힌 잡음 제거. 슬라이더 조절 시 화면 좌하단에 기준 박스를 잠깐 띄워 직관적으로 보여줌.
→ "임계값·시간·크기, 세 각도로 오탐을 깎는다." 한 노브로 안 되던 헛검출이 조합하면 사라진다.
묶음 ② 속도 레버 — 실시간성 트레이드오프
프레임 스킵 (1/N)
N프레임당 1번만 추론. FPS는 오르지만 판정이 듬성듬성. 표시 속도(재생)와 처리 속도(FPS)는 별개임을 짚는다.
입력 리사이즈 (imgsz)
모델 입력을 줄이면 빨라지고 작은 객체를 놓친다. 1교시 "큰 모델은 느리다"의 회수.
최대 속도(언락)
재생 페이싱을 풀어 가능한 한 빠르게 처리. 노브가 FPS에 주는 영향을 적나라하게 본다. (재생바 속도 버튼 옆)
묶음 ③ 위험구역(ROI) + 경보 규칙 — RISA 직결
위험구역(ROI) 그리기
화면에 폴리곤을 직접 찍어 위험구역을 지정(첫 점 다시 클릭/더블클릭=완성). 다시 그리기·지우기 지원.
진입 기준 (겹침)
박스가 ROI에 얼마나 겹쳐야 침입으로 볼지. "닿으면(any)"부터 "면적의 N%"까지. 발만 걸친 통행 vs 실제 진입을 가른다.
경보 규칙 — 인원/체류
ROI 안 인원 ≥ N명 · 체류 ≥ N초(추적 ID로 누적). 두 조건을 OR / AND로 직접 선택.
시각화 토글 + 스냅샷
박스·ID·신뢰도·ROI 표시 on/off. 경보 순간을 이미지로 저장.

멘트: "여기까지가 RISA가 현장에서 하는 일이다. 우리는 그 노브를 다 밖으로 빼놨을 뿐. 이제 여러분이 직접 맞춰본다."

16–35 핸즈온 미션 — 직접 노브를 맞춰라 실습

수강생이 같은 영상에 노브를 조합한다. 강사 설명이 아니라 손이 시간을 채우는 구간(19분). 막히면 완성본/힌트 배포로 흐름 유지.

🎯 미션 이 영상에서 헛경보 없이, 복합 규칙 — 위험구역에 동시 3명 이상 또는 한 명이라도 10초 이상 체류 — 이 제대로 울리도록 노브를 맞춰라.

왜 한 노브로는 안 되나(미션의 설계 의도):

  • 임계값만 올리면 → 진짜 사람도 놓쳐 경보가 안 울린다.
  • ROI만 그리면 → 스쳐 지나가는 통행에도 깜빡 경보(헛경보).
  • 디바운싱 + 진입 겹침 기준을 같이 써야 → "잠깐 닿음"과 "실제 체류"가 갈린다.
  • OR/AND 선택으로 → "사람이 많거나 / 오래 있거나"를 현장 규칙에 맞게 조립.

→ "오탐 3종 × 위험판정"을 조합해야 목표가 잡힌다는 걸 손으로 깨닫게 하는 게 핵심.

35–40 정리 → RISA 연결

  • 모델이 절반, 전후처리가 절반 — 같은 모델 출력도 노브 조합에 따라 쓸모가 갈린다.
  • 현장의 판단은 후처리가 만든다 — 임계값·시간·크기·구역·규칙은 전부 사람이 정한다.
  • "이 원리들이 제품 안에 세팅돼 24시간 자동으로 도는 게 RISA." (여유 시 추적/카운팅 추가 시연)

다음 복선: "보는 AI는 여기까지. 3교시는 판단하는 AI — 데이터에 직접 묻는다."

⚠ 강사용 단서 (슬라이드엔 올리지 말 것)
  • 앱 ≠ RISA를 반드시 명확히. "제품 내부를 까본 학습용 도구"로 프레이밍해야 제품 홍보톤·오해를 피한다.
  • 라이브 리스크: 앱이 강의장에서 죽을 때 대비 백업 녹화 클립 필수. 추론은 CPU(onnxruntime)라 무거운 영상은 미리 FPS 확인.
  • 기획서의 노브 셋과 실제 구현이 일부 다름 — 디바운싱은 "N프레임 연속 확인"(streak), 최소크기는 박스 높이 %, ROI 진입은 겹침 면적 비율, 경보는 OR/AND 사용자 선택. 시연·지문은 구현 기준으로.
  • 샘플 영상은 Mendeley "Safe/Unsafe Behaviours"(제조현장 고정 IP캠, CC BY 4.0) 기준. ROI·체류·카운팅에 적합한 클립을 미리 골라둘 것.
  • 수준 편차(1~3) 대응: 완성본 설정·단계별 힌트를 미리 배포, 빠른 수강생용 심화(예: AND로 바꿔 더 까다롭게 / 진입 겹침을 올려 통행 무시).

내부 논의 포인트

  • 투어 11분(5–16)에 노브 9개가 빡빡하지 않은가 — 묶음 ①②는 빠르게, ③(ROI/경보)에 시간을 몰아줄지.
  • 미션 영상 확정: 동시 3명·10초 체류가 자연스럽게 나오는 클립이 있는가, 아니면 미션 수치를 영상에 맞춰 조정할지.
  • 수강생 배포 방식: 실행파일(PyInstaller) vs 강사 화면 시연 + 일부만 직접. onefile 빌드 시 QtSvg·qdarktheme 데이터 포함 필요(이미 식별됨).
  • 1교시 회수 포인트(임계값·큰 모델은 느리다·분포 밖 실패→밝기/대비)를 어디까지 명시적으로 연결할지.
